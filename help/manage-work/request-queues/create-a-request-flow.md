# 在Workfront中建立請求流程

>[!PREREQUISITES]
>
>* [在Workfront中建立請求佇列](/help/manage-work/request-queues/create-a-request-queue.md)
>* [組織您的要求流程設定](/help/manage-work/request-queues/understand-settings-for-a-flow-request.md)

此影片提供逐步指南，說明如何設定佇列主題、主題群組及路由規則，以建立請求流程。流&#x200B;程從建立路由規則開始，路由規則定義指派請求的方式&#x200B;。接著，會建立主題群組來組織請求&#x200B;。這些群組可新增至佇列，而且可建立的主題群組數目沒有限制。
然後影片會示範如何建立佇列主題，這些主題會連結至路由規則、主題群組、自訂表單、核准程式和預設持續時間。
最後，影片會說明如何透過提交請求來測試設定，以確保其正確路由。如&#x200B;果發生任何問題，可對佇列進行調整。影片&#x200B;強調驗證請求流程結構和功能的重要性，以確保正確的路由和組織。

>[!VIDEO](https://video.tv.adobe.com/v/3433833/?captions=chi_hant&quality=12&learn=on&enablevpops=0)

## 重點提要

* **路由規則設定：**&#x200B;路由規則會決定如何指派要求，不論是指派給團隊（例如Creative團隊）或個人（例如Jennifer Campbell或Mark Lewis）。 這&#x200B;些規則是透過命名、新增說明以及選取預設受指派人或專案團隊來建立的。
* **主題群組建立：**&#x200B;主題群組可協助您將請求組織成「數位」和「列印」等類別。 可&#x200B;建立的主題群組數量沒有限制，最多可巢狀內嵌10層深。
* **佇列主題組態：**&#x200B;佇列主題連結到路由規則、主題群組、自訂表單、核准程式和預設期間。 例&#x200B;如，名為「手冊」的佇列主題可與列印主題群組相關聯，並路由至Creative團隊。
* **測試要求流程：**&#x200B;設定要求佇列後，必須透過提交要求來測試流程，以確保路由及功能正確無誤。任何&#x200B;問題都可透過重新造訪佇列設定來解決。&#x200B;URL
* **調整的彈性：**&#x200B;系統允許修改路由規則、主題群組和佇列主題（如果有任何內容看起來不妥當），以確保請求流程保持有效率且準確。
