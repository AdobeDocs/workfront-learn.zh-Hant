# 組織您的請求流程設定

影片說明Workfront中的請求流程程式，重點是佇列管理、路由規則、主題群組和佇列主題。 它&#x200B;會回答三個關鍵問題，強調組織請求佇列的重要性：

1. **我需要哪些主題？**  — 這&#x200B;些是佇列主題，可分類提交的請求，使其更易於管理和自訂。 
1. **我可以進一步組織我的主題嗎？** -&#x200B;主題群組可協助將相關的佇列主題分組，以便進行更好的組織（例如，手冊、名片和「列印」底下的廣告牌）。 
1. **應該將要求指派給誰？** -&#x200B;路由規則會自動將提交內容導向特定團隊、角色或使用者，確保不會忽略請求。 

透過解決這些問題，使用者就能在Workfront中有效地規劃及設定要求佇列。 &#x200B;

>[!VIDEO](https://video.tv.adobe.com/v/335222/?quality=12&learn=on&enablevpops=1)

## 重點提要

* **佇列主題：**&#x200B;定義並將不同型別的請求組織到佇列主題中，以簡化分類和自訂。 &#x200B;
* **主題群組：**&#x200B;使用主題群組來進一步組織佇列主題，讓您更容易找到相關請求（例如，將「列印」和「數位」類別分組）。 &#x200B;
* **路由規則：**&#x200B;設定路由規則，以自動將請求指派給特定團隊、角色或使用者，避擴音交內容遭到忽略。 &#x200B;
* **規劃是必要的：**&#x200B;在建置要求佇列之前，先組織要求佇列的需求，以確保有效率的管理。 &#x200B;
* **避免未注意的請求：**&#x200B;若沒有路由規則，提交可能會保留在問題標籤中，除非定期手動檢查，否則不會注意到。 &#x200B;
