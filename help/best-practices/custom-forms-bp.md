# 最佳實務 — 自訂表單

## 什麼是Adobe Workfront的「最佳實務」？

最佳做法是代表有效、有效行動方針的准則；易於被您和公司用戶採納；並可在貴組織中成功複製。

在您檢閱這些建議時，請記住，有些Workfront最佳實務是通用的，有些則可能是更具體的主題。 將這些最佳實務作為架構，協助您指導Workfront系統的設定與使用。

## 瀏覽此頁面

當您捲動此頁面時，首先會找到主題所有最佳實務的高階清單。 這可讓您檢閱建議，而不需深入了解「原因」的詳細資訊。

「這些最佳實務為何？」 在高階清單後找到的「 」區域，會更詳細說明某些最佳實務，以及為何這些作法被視為程式、工具等，您應考慮使用Workfront執行個體實作。

</br>
</br>

## 自訂表單最佳實務

* 在Adobe Workfront中建立自訂表單之前，先在白板或紙張上繪製表單圖。

* 如果使用該範本製作的專案一律需要附加特定自訂表單，請將自訂表單附加至專案範本。 您甚至可以填寫很少變更或需要特定資訊的欄位。

* 建立自訂表單以追蹤您以公司身分新增至Workfront系統的外部客戶或廠商的相關資訊。

* 建立包含通用欄位和計算欄位的單一「一般」或「通用」自訂表單，而非將個別欄位追蹤至多個表單。 這有助於您以一致的方式收集整個企業的資訊。

* 在自訂表單中新增分段，以讓表單保持井然有序且易於理解。

* 縮短自訂表單，確保表單已完全填寫，而且您會獲得所需的所有資訊。

* 使用預定義的自定義欄位選項（如單選按鈕和下拉菜單），通過要求用戶從特定選項中選擇來限制答案選項。

* 請確定欄位標籤的措詞清晰、描述性強。

* 在自訂欄位的「指示」欄位中新增資訊，以指出填寫欄位的人員必須輸入哪些資訊。

* 以特定方式填入其他欄位時，請使用顯示邏輯來顯示所需欄位。 在自訂表單中使用略過邏輯來隱藏與請求類型無關的欄位。

* 從其他表單中提取計算資訊，減少自訂表單所需的計算數量。

* 為自訂表單建立新欄位之前，請先檢查欄位庫，查看欄位是否已存在。

* 使用必填欄位，確保捕獲對您的流程或組織至關重要的資訊。

* 在自訂表單上變更欄位名稱時請務必小心，因為這會影響呼叫該欄位的計算欄位。

* 定期檢閱自訂表單和欄位（例如每季一次），作為定期Workfront系統維護的一部分。

* 停用不需要的自訂表單，而非刪除表單。

* 隱藏區段分隔下自訂表單上不再需要的自訂欄位。 然後，讓該區段只對系統管理員可見。

* 限制可在您的Workfront例項中建立自訂表單的使用者。

## 這些最佳實務為何？

**最佳實務**

在Adobe Workfront中建立自訂表單之前，先在白板或紙張上繪製表單圖。

**原因如下**

請確定您有各欄位，可填寫所有必要資訊，而且欄位的組織方式可讓使用者輕鬆填寫表單。 對表單進行圖示也有助於您判斷是否可使用顯示邏輯來隱藏/顯示某些欄位。

</br>
</br>

**最佳實務**

如果使用該範本製作的專案一律需要附加特定自訂表單，請將自訂表單附加至專案範本。 您甚至可以填寫很少變更或需要特定資訊的欄位。

**原因如下**

這樣，表單即已附加，部分資訊已填寫，有助於加快專案建立速度，並確保所有適用專案中的這些自訂欄位都已正確且完整地填寫。

</br>
</br>

**最佳實務**

建立自訂表單以追蹤您以公司身分新增至Workfront系統的外部客戶或廠商的相關資訊。

**原因如下**

使用自訂表單來追蹤地址、主要連絡人姓名等，以便在Workfront中輕鬆存取。 您也可以將此自訂表單資訊提取至報表中。

</br>
</br>

**最佳實務**

建立包含通用欄位和計算欄位的單一「一般」或「通用」自訂表單，而非將個別欄位追蹤至多個表單。 這有助於您以一致的方式收集整個企業的資訊。

**原因如下**

「一般」表單可讓維護這些欄位更輕鬆，因為這些欄位都生活在一個地方。 您將能更新單一表單，而不是讓這些個別欄位以數種不同表單存在，之後您必須個別更新。

</br>
</br>

**最佳實務**

在自訂表單中新增分段，以讓表單保持井然有序且易於理解。

**原因如下**

將相關資訊分組為章節，可協助使用者導覽表單。

</br>
</br>

**最佳實務**

縮短自訂表單，確保表單已完全填寫，而且您會獲得所需的所有資訊。

**原因如下**

長表單對使用者而言可能令人望而生畏，且經常導致表單無法完整填寫。 這反過來又會導致分配資訊不完整和報告資料不準確。

如果自訂表單有許多欄位，請盡可能並排放置相關欄位，讓使用者的捲動次數減少。 您也可以使用略過邏輯來隱藏不需要填寫的欄位，或顯示邏輯以顯示特定欄位。

</br>
</br>

**最佳實務**

使用預定義的自定義欄位選項（如單選按鈕和下拉菜單），通過要求用戶從特定選項中選擇來限制答案選項。

**原因如下**

使用預先定義的欄位，使用者只需按一下方塊或從功能表中選取，即可確保該問題的所有答案相同。

一致、準確的資料對於準確的報告至關重要。 資料不一致會導致報表不準確，進而影響個別層級的決策。 此外，此一致的資料可讓您將圖表新增至報表，以視覺化呈現資料。 不能在圖表中使用開放式文本欄位。

</br>
</br>

**最佳實務**

請確定欄位標籤的措詞清晰、描述性強。

**原因如下**

這樣，填寫自訂表單的人員就能了解要求提供哪些資訊。

</br>
</br>

**最佳實務**

在自訂欄位的「指示」欄位中新增資訊，以指出填寫欄位的人員必須輸入哪些資訊。

**原因如下**

當您將游標暫留在？ 自訂表單欄位旁的圖示。 包括應在欄位中輸入的資料，包括文本欄位的任何必需格式。

為使用者提供更詳細的資訊有助於減少額外的對話、來回寄送電子郵件或混淆。 當資訊不完整或遺失時，可能會延遲完成工作。

</br>
</br>

**最佳實務**

以特定方式填入其他欄位時，請使用顯示邏輯來顯示所需欄位。 在自訂表單中使用略過邏輯來隱藏與請求類型無關的欄位。

**原因如下**

只顯示所需欄位，或隱藏不需要的欄位，可簡化自訂表單，減少使用者填寫自訂表單時的混淆。 這也有助於整體縮短表單，減少對使用者的恐嚇，並提升較高的回應率。

使用顯示邏輯也可協助您減少建立和維護所需的自訂表單數量。

</br>
</br>

**最佳實務**

從其他表單中提取計算資訊，減少自訂表單所需的計算數量。

**原因如下**

例如，問題表單上有一個名為「資產數量」的計算欄位，可計算附加至項目的資產數量。 這會用於請求佇列。 轉換請求時，您需要此資訊才能傳遞至專案。 製作問題表單副本並儲存為專案表單。 然後將問題計算欄位的名稱添加到項目表單的計算框中。 在此範例中，在專案表單的計算欄位（稱為「資產數量」）中，按字面輸入「資產數量」。 這可防止Workfront嘗試在專案上重新執行此計算，而是使用問題自訂表單中的值。

</br>
</br>

**最佳實務**

如果適用，請使用欄位庫中提供相同用途的現有欄位。

**原因如下**

Workfront中的兩個欄位不能有相同的名稱。 如果欄位有共同名稱，則該欄位可能已存在。 在建立新欄位之前，請檢查欄位庫，以查看欄位是否已存在。

使用您未建立的欄位時，請記住，對此欄位的修改會影響「所有自訂表單」，此欄位是的一部分。 如果需要更改計算或更改欄位類型（例如，文本改為單選按鈕），則需要建立新欄位，並為其指定與原始欄位不同的名稱。 （請記得，使用者可能會混淆名稱相似的多個欄位。）

</br>
</br>

**最佳實務**

使用必填欄位，確保捕獲對您的流程或組織至關重要的資訊。

**原因如下**

自訂表單上的資料不完整可能會延遲工作，並影響報表。 必填欄位指標（欄位名稱旁的紅色*）會提醒使用者在編輯、儲存該自訂表單或正式提交請求之前，需要特定資訊。

但是，必填欄位應謹慎謹慎使用。 要求每個欄位，可能會讓使用者無法使用有用且完整的資訊填寫欄位。 此外，在對象的「詳細資訊」區域中編輯自定義表單時，不完整的必需欄位將阻止保存自定義表單。

</br>
</br>

**最佳實務**

在自訂表單上變更欄位名稱時請務必小心，因為這會影響呼叫該欄位的計算欄位。

**原因如下**

當您變更欄位名稱時，需要更新名稱，該名稱用於自訂表單上的計算自訂欄位，或內建至文字模式的計算中。 更改欄位名稱可能會中斷計算，並導致資訊不準確。

</br>
</br>

**最佳實務**

定期檢閱自訂表單和欄位（例如每季一次），作為定期Workfront系統維護的一部分。

**原因如下**

如果自訂表單和欄位未收集您組織完成工作所需的資料，則這些表單和欄位將沒有用。

進行更新時，請留意這些變更對Workfront其他方面的影響。 例如，變更欄位名稱可能會中斷使用該欄位的計算。 或者，變更表單的標籤或名稱可能會導致報表無法顯示所需的資訊，或與其他系統的整合無法執行。

除了更新自訂表單外，請識別使用量低或完全未使用的表單。 不常使用的表單是否可與其他表單結合？ 或者，該不再使用該表單了，因為您的團隊不再需要該表單收集的資訊。

在對這些不必要的表單執行任何動作之前，請先參閱停用與刪除自訂表單和自訂欄位的相關資訊。

</br>
</br>

**最佳實務**

停用不需要的自訂表單，而非刪除表單。

**原因如下**

刪除自訂表單時，會刪除透過該自訂表單輸入的所有自訂資料。
停用自訂表單會保留所有相關的歷史資料。 這表示您可以繼續報告此資訊。

停用也會防止表單顯示在下拉式功能表中，供使用者選取自訂表單。 不過，表單仍會顯示在已附加至的任何物件上。

</br>
</br>

**最佳實務**

隱藏區段分隔下自訂表單上不再需要的自訂欄位。 然後，讓該區段只對系統管理員可見。

**原因如下**

刪除現有的自訂欄位不僅會從自訂表單中移除該欄位，而且會刪除該欄位中包含的所有資料（無論使用該自訂欄位的位置）。 這表示會刪除報表可能需要的歷史資料。

為防止資料遺失，請隱藏自訂表單本身中不需要的欄位。 自訂表單中的區段分段可讓您根據使用者是否擁有檢視、貢獻或管理其附加至的Workfront物件，來隱藏或公開該區段中屬於該區段的欄位。 或者，區段可設為「僅管理員」，這樣只有具有系統管理員存取層級的使用者才能看到表單的整個區段。

</br>
</br>

**最佳實務**

限制可在您的Workfront例項中建立自訂表單的使用者。


**原因如下**

擁有可建立自訂表單的特定人員群組，有助於控制在Workfront例項中建立的自訂表單數量。

此外，允許其他人建立表單可以減輕系統管理員的工作負擔，並讓每個群組能夠控制他們經常使用的自訂表單更新。
