# 最佳實務 — 自訂表單

## 什麼是Adobe Workfront「最佳實務」？

最佳實務准則是代表有效率、有效率的行動方針；您和您公司的使用者可輕鬆採用；並且可在您的組織內成功複製。

在檢閱這些建議時，請記住，部分Workfront最佳實務是通用的，而其他實務則可能更特定於主題。 使用這些最佳實務作為框架，以協助指導您的Workfront系統設定和使用。

## 瀏覽此頁面

捲動瀏覽本頁面時，首先您會找到該主題所有最佳實務的高層級清單。 這可讓您檢閱建議，而不需深入瞭解「原因」的詳細資訊。

「為什麼這些是最佳實務？」 區域位於高階清單之後，可讓您更詳細地瞭解部分最佳實務，以及為何將其視為流程、工具等，因此您應考慮使用Workfront執行個體進行實作。

</br>

## 自訂表單最佳實務

* 在Adobe Workfront中建立自訂表單之前，先在白板或紙張上繪製圖表。

* 如果使用該範本建立的專案一律需要附加某些自訂表單，請將自訂表單附加至專案範本。 您甚至可以填寫很少變更或需要特定資訊的欄位。

* 建立自訂表單，以追蹤您作為公司新增到Workfront系統的外部客戶或廠商的相關資訊。

* 建立包含通用欄位和已計算欄位的單一「一般」或「通用」自訂表單，而不是將個別欄位附加到多個表單上。 這可協助您以一致的方式收集企業範圍的資訊。

* 在自訂表單中新增分割槽符號，使其保持井然有序，易於理解。

* 縮短自訂表格，以確保表格已完整填寫，且您會獲得所需的所有資訊。

* 使用預先定義的自訂欄位選項（例如選項按鈕和下拉式選單），透過要求使用者從特定選項中進行選取來限制回答選項。

* 請確定欄位標籤的措辭清楚且具描述性。

* 在自訂欄位的「指示」欄位中新增資訊，以指示填寫欄位的人必須輸入哪些資訊。

* 以特定方式填入其他欄位時，請使用顯示邏輯來顯示所需欄位。 在自訂表單中使用略過邏輯來隱藏與請求型別無關的欄位。

* 從其他表單提取計算資訊，減少自訂表單所需的計算數量。

* 在為自訂表單建立新欄位之前，請檢查欄位庫，以檢視該欄位是否已存在。

* 使用必要欄位，以確保擷取對您的流程或組織至關重要的資訊。

* 變更自訂表單上的欄位名稱時請務必小心，因為這會影響呼叫該欄位的計算欄位。

* 定期審查自訂表單和欄位（例如每季一次），作為定期Workfront系統維護的一部分。

* 停用不需要的自訂表單，而不是刪除它們。

* 在分割槽符號下隱藏自訂表單上不再需要的自訂欄位。 然後讓系統管理員看見該區段。

* 限制可在您的Workfront執行個體中建立自訂表單的使用者。

## 為什麼有這些最佳實務？

**最佳實務**

在Adobe Workfront中建立自訂表單之前，先在白板或紙張上繪製圖表。

**原因如下**

請確定您有欄位可取得所有必要資訊，且欄位的組織方式可讓使用者輕鬆填寫表單。 對圖表化表單也可協助您判斷是否可使用顯示邏輯來隱藏/顯示某些欄位。

</br>
</br>

**最佳實務**

如果使用該範本建立的專案一律需要附加某些自訂表單，請將自訂表單附加至專案範本。 您甚至可以填寫很少變更或需要特定資訊的欄位。

**原因如下**

如此一來，表單已附加且部分資訊已填寫，這有助於加快專案建立速度並確保這些自訂欄位在所有適用專案中正確且完整地填寫。

</br>
</br>

**最佳實務**

建立自訂表單，以追蹤您作為公司新增到Workfront系統的外部客戶或廠商的相關資訊。

**原因如下**

使用自訂表單來追蹤地址、主要連絡人姓名等，以便在Workfront中輕鬆存取。 您也可以將此自訂表單資訊提取到報表中。

</br>
</br>

**最佳實務**

建立包含通用欄位和已計算欄位的單一「一般」或「通用」自訂表單，而不是將個別欄位附加到多個表單上。 這可協助您以一致的方式收集企業範圍的資訊。

**原因如下**

「一般」表單可讓維護這些欄位變得更輕鬆，因為它們都集中在一處。 您將能夠更新單一表單，而不是讓這些個別欄位存在於數個不同的表單中，然後您必須個別更新。

</br>
</br>

**最佳實務**

在自訂表單中新增分割槽符號，使其保持井然有序，易於理解。

**原因如下**

將相關資訊分組為區段可協助使用者導覽表單。

</br>
</br>

**最佳實務**

縮短自訂表格，以確保表格已完整填寫，且您會獲得所需的所有資訊。

**原因如下**

長表格可能會讓使用者望而卻步，並且經常導致表格無法完整填寫。 這進而導致指派資訊不完整，且報告的資料不準確。

如果自訂表單有很多欄位，請儘可能將相關欄位並排放置，以減少使用者的捲動次數。 您也可以使用略過邏輯來隱藏不需要填寫的欄位，或使用顯示邏輯來顯示特定欄位。

</br>
</br>

**最佳實務**

使用預先定義的自訂欄位選項（例如選項按鈕和下拉式選單），透過要求使用者從特定選項中進行選取來限制回答選項。

**原因如下**

有了預先定義的欄位，使用者只需按一下方塊或從選單中選取，即可確保該問題的所有答案都相同。

一致、準確的資料對於準確的報告至關重要。 資料不一致會導致報告不正確，這可能會影響從個人到個人的決策。 此外，此一致的資料可讓您新增圖表至報表，以視覺化呈現您的資料。 開放式文字欄位不能在圖表中使用。

</br>
</br>

**最佳實務**

請確定欄位標籤的措辭清楚且具描述性。

**原因如下**

如此一來，填寫自訂表格的人就能瞭解需要哪些資訊。

</br>
</br>

**最佳實務**

在自訂欄位的「指示」欄位中新增資訊，以指示填寫欄位的人必須輸入哪些資訊。

**原因如下**

當您將滑鼠游標停留在上方時，此資訊會顯示為快顯視窗？ 自訂表單上欄位旁的圖示。 包括應在欄位中輸入哪些資料，包括文字欄位的任何必要格式。

為使用者提供更多詳細資訊，有助於減少其他交談、來回電子郵件或混淆。 當資訊不完整或遺失時，可能會延遲完成的工作。

</br>
</br>

**最佳實務**

以特定方式填入其他欄位時，請使用顯示邏輯來顯示所需欄位。 在自訂表單中使用略過邏輯來隱藏與請求型別無關的欄位。

**原因如下**

僅顯示所需的欄位或隱藏不需要的欄位，可建立更乾淨的自訂表單，並減少使用者填寫自訂表單時的混淆。 這也有助於縮短表單的整體長度，減少使用者的恐嚇性，並提升回應率。

使用顯示邏輯也可協助您減少需要建立和維護的自訂表單數量。

</br>
</br>

**最佳實務**

從其他表單提取計算資訊，減少自訂表單所需的計算數量。

**原因如下**

例如，在問題表單上有一個名為「資產數量」的計算欄位，該欄位會計算附加到專案的資產數量。 這會用於請求佇列。 在轉換請求時，您需要此資訊以延續到專案。 製作問題表單的副本並將其另存為專案表單。 然後將問題計算欄位的名稱新增到專案表單的計算方塊中。 在此範例中，在專案表單中名為「資產數目」的計算欄位中，按字面含義輸入「資產數目」。 這可防止Workfront嘗試在專案上重新執行此計算，而是使用問題自訂表單中的值。

</br>
</br>

**最佳實務**

適用時，使用欄位庫中用途相同的現有欄位。

**原因如下**

Workfront中的兩個欄位不能有相同的名稱。 如果欄位具有通用名稱，則該欄位可能已存在。 在建立新欄位之前檢查欄位程式庫，以檢視欄位是否已存在。

使用您未建立的欄位時，請記住，對此欄位的修改會影響該欄位所屬的所有自訂表單。 如果您需要變更計算或變更欄位型別（例如文字變成選項按鈕），則需要建立新欄位，並為它指定與原始欄位不同的名稱。 （請記住，名稱相似的多個欄位可能會讓使用者感到困惑。）

</br>
</br>

**最佳實務**

使用必要欄位，以確保擷取對您的流程或組織至關重要的資訊。

**原因如下**

自訂表單上的資料不完整可能會延遲工作並影響報告。 必要欄位指標（欄位名稱旁的紅色*）會提醒使用者，在編輯然後儲存該自訂表單或正式提交請求之前，需要特定資訊。

不過，必要欄位應謹慎使用。 擁有每個必填欄位可能會讓使用者無法在欄位中填入有用且完整的資訊。 此外，在物件的「詳細資訊」區域中編輯自訂表單時，不完整的必填欄位會阻止儲存自訂表單。

</br>
</br>

**最佳實務**

變更自訂表單上的欄位名稱時請務必小心，因為這會影響呼叫該欄位的計算欄位。

**原因如下**

當您變更欄位名稱時，需要更新名稱，以在自訂表單的計算自訂欄位或內建在文字模式的計算中使用它。 變更欄位名稱可能會中斷計算，並導致不正確的資訊。

</br>
</br>

**最佳實務**

定期審查自訂表單和欄位（例如每季一次），作為定期Workfront系統維護的一部分。

**原因如下**

如果自訂表單和欄位未收集您的組織完成工作所需的資料，則這些表單和欄位沒有用處。

進行更新時，請留意這些變更對Workfront其他方面有何影響。 例如，變更欄位名稱可能會中斷使用該欄位的計算。 或變更表單的標籤或名稱，可能導致報表不顯示所需資訊，或導致與其他系統的整合無法執行。

除了更新自訂表單之外，請識別使用率低或完全沒有使用的表單。 未經常使用的表單可以與其他表單結合嗎？ 或許，該表單是時候被取代了，因為您的團隊不再需要表單正在收集的資訊。

在使用這些不必要的表單採取任何動作之前，請先閱讀有關停用與刪除自訂表單和自訂欄位的資訊。

</br>
</br>

**最佳實務**

停用不需要的自訂表單，而不是刪除它們。

**原因如下**

當您刪除自訂表單時，也會刪除透過該自訂表單輸入的所有自訂資料。
停用自訂表單會保留所有關聯的歷史資料。 這表示您可以繼續報告此資訊。

停用也會防止表單顯示在使用者選取自訂表單的下拉式功能表中。 不過，表單仍會顯示在其已附加的任何物件上。

</br>
</br>

**最佳實務**

在分割槽符號下隱藏自訂表單上不再需要的自訂欄位。 然後讓系統管理員看見該區段。

**原因如下**

刪除現有自訂欄位不僅會將其從自訂表單中移除，而且會刪除該欄位中包含的所有資料，無論該自訂欄位在何處使用。 這表示刪除報表可能需要的歷史資料。

若要防止資料遺失，請隱藏自訂表單本身中不需要的欄位。 自訂表單中的分割槽符號可讓您根據使用者是否有權檢視、貢獻或管理其所附加的Workfront物件，隱藏或公開屬於該分割槽一部分的欄位。 或者，區段可以設為「僅限管理員」，因此只有具有系統管理員存取層級的使用者才能檢視表單的整個區段。

</br>
</br>

**最佳實務**

限制可在您的Workfront執行個體中建立自訂表單的使用者。


**原因如下**

選擇一組可以建立自訂表單的人員，這有助於控制在Workfront執行個體中建立的自訂表單數量。

此外，允許其他人建立表單可以減輕系統管理員的工作，並讓每個群組都能控制他們定期使用的自訂表單的更新。
