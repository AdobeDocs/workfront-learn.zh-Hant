# 最佳實務 — 授權和存取層級

## 什麼是Adobe Workfront「最佳實務」？

最佳實務准則是代表有效率、有效率的行動方針；您和您公司的使用者可輕鬆採用；並且可在您的組織內成功複製。

在檢閱這些建議時，請記住，部分Workfront最佳實務是通用的，而其他實務則可能更特定於主題。 使用這些最佳實務作為框架，以協助指導您的Workfront系統設定和使用。

## 瀏覽此頁面

捲動瀏覽本頁面時，首先您會找到該主題所有最佳實務的高層級清單。 這可讓您檢閱建議，而不需深入瞭解「原因」的詳細資訊。

「為什麼這些是最佳實務？」 區域位於高階清單之後，可讓您更詳細地瞭解部分最佳實務，以及為何將其視為流程、工具等，因此您應考慮使用Workfront執行個體進行實作。

</br>

## 授權和存取層級最佳實務

* 設定存取層級時，首先減少使用者的存取許可權。

* 在指派「檢閱和請求」授權時，通常會預設為「檢閱」，因為它在Adobe Workfront中為使用者提供了更多許可權。

* 取消勾選所有存取層級中每個物件的「共用系統範圍」核取方塊，除非您明確希望這些使用者能夠執行。

* 請考慮在存取層級中設定其他限制底下啟用「永不允許使用者刪除評論」設定。

* 限制系統管理員人數，以支援群組管理員。

* 複製現有的存取層級並進行修改，而不是從頭開始建立新的存取層級。

* 在「說明」方塊中記錄每個存取層級的功能。

* 將您自己限制在達成工作目標所需的存取層級，最好是四或五個，以符合系統中大多數使用者的需求。

* 至少指派兩名使用者為全域系統管理員存取層級。

* 限制使用者透過共用可對Workfront專案執行的操作，而不是移除存取層級中的功能。

</br>
</br>


## 為什麼有這些最佳實務？

**最佳實務**

設定存取層級時，首先減少使用者的存取許可權。



**原因如下**

以使用者完成工作所需的最低存取權開始使用者。 如果他們因存取許可權不足而無法執行工作，通常會請求額外的存取許可權。 立即授予使用者太多存取權可能導致安全性問題。 此外，給予使用者更多存取權總比剝奪使用者存取權要好。

</br>
</br>



**最佳實務**

在指派「檢閱和請求」授權時，通常會預設為「檢閱」，因為它在Adobe Workfront中為使用者提供了更多許可權。



**原因如下**

雖然「檢閱」和「請求」授權均可指派給Workfront中的無限數量使用者，「請求」授權卻僅限於提出和更新請求。 檢閱授權比請求授權更能存取專案和任務，以及檢視投資組合和計畫、編輯檔案和存取資源管理工具的可能性。

</br>
</br>

**最佳實務**

取消勾選您所有存取層級中每個物件的「共用系統範圍」核取方塊，除非使用者有特定原因需要能夠共用系統範圍。



**原因如下**

在系統範圍內共用物件通常可作為運送工具，讓特定使用者檢視Workfront中的專案。 缺少Workfront群組結構或無法完全瞭解共用許可權時，就會發生這種情況。 在系統範圍內共用專案時，這表示每個人都可以看到共用的專案。 根據系統中保留的資訊型別，這可能會導致隱私權問題。



例如，您可能正在與Workfront內的多家廠商合作，以檢查進度、提供核准等。 如果「共用系統範圍」核取方塊是一個選項，可選取或設定為預設值，讓所有廠商都能取得資訊。



完全取消核取選項後，您便可讓擁有共用許可權的使用者決定要與其共用物件的特定人員或人員（透過公司、群組或團隊）。

</br>
</br>

**最佳實務**

請考慮在存取層級中設定其他限制底下啟用「永不允許使用者刪除評論」設定。



**原因如下**

啟用此選項可確保不會從Workfront移除過去的通訊。 某些組織會要求保留完整的註解歷史記錄以進行稽核。

</br>
</br>

**最佳實務**

限制系統管理員人數，以支援群組管理員。



**原因如下**

系統管理員可以存取Workfront中的所有專案，包括全域系統設定。 系統管理員可以存取的設定群組管理員是由系統管理員所控制，且僅套用至該特定群組。



擁有群組管理員可讓系統管理員委派許多責任，讓他們專注於更大的專案，而不是Workfront的日常維護。 群組管理員可更輕鬆地與群組的需求保持聯絡，進而為使用者提供更好的服務。

</br>
</br>


**最佳實務**

複製現有的存取層級並進行修改，而不是從頭開始建立新的存取層級。



**原因如下**

複製現有存取層級可為新存取層級提供一致的基礎，確保初始設定完全相同。 這也有助於節省時間，因為系統管理員不必從頭開始完全設定存取層級。

</br>
</br>

**最佳實務**

在「說明」方塊中記錄每個存取層級的功能。



**原因如下**

詳細說明並列出每種物件型別的設定。 這可協助目前和未來的系統管理員確切瞭解每個存取層級的功能，而不必深入瞭解存取層級本身來檢閱設定。



這也使得在報告中檢視存取層級時比較存取層級更容易。 說明欄位可以快速新增到檢視中，以快速檢視它們之間的差異以及可能建立不同存取層級的原因。

</br>
</br>

**最佳實務**

將您自己限制在達成工作目標所需的存取層級，最好是四或五個，以符合系統中大多數使用者的需求。


**原因如下**

存取層級可確保在將Workfront物件與使用者共用時，使用者擁有編輯、刪除物件等所需的許可權。 您可以將存取層級設為更一般，因為個別專案上的共用可設定為更具體。


此外，存取層級較少可讓系統維持整齊且易於實作策略，當人員加入公司或切換部門時，也可以讓入門更快。

</br>
</br>

**最佳實務**

至少指派兩名使用者為全域系統管理員存取層級。

**原因如下**

多名人員應瞭解為何會依原樣設定Workfront、如何管理/維護，以及如何支援使用者。 如果有人不在辦公室、離開組織、忙碌等，這可以確保有另一個人擁有成功管理系統所需的資訊和知識。

</br>
</br>

**最佳實務**

限制使用者透過共用可對Workfront專案執行的操作，而不是移除存取層級中的功能。


**原因如下**

存取層級會控制使用者可以對全域層級上的特定專案執行的操作。 每個專案、任務、專案組合、檔案等的共用許可權可控制個別使用者可對該特定專案執行的操作。 與其移除具有特定存取層級之所有人的功能，不如微調特定專案的共用許可權，讓使用者擁有有限的控制權。
