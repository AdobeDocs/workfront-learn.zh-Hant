# 最佳實務 — 專案範本

## 什麼是Adobe Workfront「最佳實務」？

最佳實務准則是代表有效率、有效率的行動方針；您和您公司的使用者可輕鬆採用；並且可在您的組織內成功複製。

在檢閱這些建議時，請記住，部分Workfront最佳實務是通用的，而其他實務則可能更特定於主題。 使用這些最佳實務作為框架，以協助指導您的Workfront系統設定和使用。

## 瀏覽此頁面

捲動瀏覽本頁面時，首先您會找到該主題所有最佳實務的高層級清單。 這可讓您檢閱建議，而不需深入瞭解「原因」的詳細資訊。

「為什麼這些是最佳實務？」 區域位於高階清單之後，可讓您更詳細地瞭解部分最佳實務，以及為何將其視為流程、工具等，因此您應考慮使用Workfront執行個體進行實作。

</br>
</br>

## 專案範本最佳實務

* 建立專案時使用範本。

* 建立專案範本的命名慣例。

* 建立可以製作和更新專案範本的選定使用者群組。

* 在專案範本上使用專案共用可自動授予使用該範本建立的專案存取權。

* 將職位角色或團隊指派給任務，而不是個人。

* 在專案範本中建立任務時，請避免變得太精細。 將專案範本中的任務數限製為完成工作所需的任務數。

* 使用任務說明來擷取任務的小型步驟，而不是將其劃分為多個任務。

* 請確定範本任務包含任務持續時間、計畫時數和前置任務。

* 預先設定專案詳細資訊，並在範本上附加自訂表單。

* 定期檢閱和更新專案範本。

* 檢查範本，確保它們擁有所有需要的資訊，然後您才能共用範本，讓其他人開始使用。

* 當您調整範本上的「排程起始日期」選項時，請複查並更新任務限制。

* 檢查範本上的專案團隊，並移除不會與專案相關聯的使用者。

## 為什麼有這些最佳實務？

**最佳實務**

建立專案時使用範本。

**原因如下**

專案範本讓專案經理（和其他建立專案的人）不必再猜測專案應包含哪些任務、如何建構時間表等。 範本是加速專案建立的最有效方式。

重要的是，範本可跨類似型別的專案提供一致性，以便每次都以相同的方式詳細說明人員、流程和資料點。 即使是快速週轉（一或兩天）和最小任務的專案，也可以受益於使用專案範本建立。

跨專案的一致性可產生更準確的資料，這對您團隊、部門以及整個組織進行決策至關重要。

</br>
</br>

**最佳實務**

建立專案範本的命名慣例。

**原因如下**

一致的命名讓範本更容易找到。 此外，當多個團隊或部門間有名稱相似的範本時，它也可以幫助專案經理和其他建立專案的人選取正確的範本。

</br>
</br>

**最佳實務**

建立可以製作和更新專案範本的選定使用者群組。

**原因如下**

擁有建構良好、一致的專案範本是良好工作管理和準確報告的關鍵。 限制可編輯範本的使用者人數，以避免意外或未經核准的變更。

</br>
</br>

**最佳實務**

在專案範本上使用專案共用可自動授予使用該範本建立的專案存取權。

**原因如下**

特定專案的存取權會透過專案本身授予。 如果同一群組的人永遠都需要存取使用特定範本建立的專案，請在範本的「專案共用」選項下新增他們。 您不僅可以在專案建立後立即控制對專案的存取，還可在未來需要變更許可權時簡化擴充性的工作。

**注意**：範本共用會授予範本本身的存取權。 使用者必須至少具有檢視許可權，才能使用範本製作專案。

</br>
</br>

**最佳實務**

將職位角色或團隊指派給任務，而不是個人。

**原因如下**

當個別使用者變更職位或離開組織時，您必須手動更新包含該人員的專案範本。 這需要系統或群組管理員或專案經理的時間。

如果您在樣版上使用職務角色或專案團隊，人員配置變更將不會對您的專案樣版產生直接影響，因為任何被指派該職務角色或該專案團隊的人都可以被指派該工作。 這有助於確保工作不會滑過裂縫。 工作角色指派也可讓指派工作給個別使用者變得更輕鬆，因為Workfront可向您顯示指派該工作角色的人員清單。

此外，Workfront的資源計畫工具會使用職務角色，協助您計算所需資源，並規劃未來工作。

</br>
</br>

**最佳實務**

在專案範本中建立任務時，請避免變得太精細。 將專案範本中的任務數限製為完成工作所需的任務數。

**原因如下**

過於複雜的專案範本會給使用者（專案經理、資源經理、團隊成員等）帶來不良體驗。 任務過多導致專案時間表難以管理，因為任務截止日期重疊，且多個任務被指派給相同職位角色或個人。


</br>
</br>

**最佳實務**

使用任務說明來擷取任務的小型步驟，而不是將其劃分為多個任務。

**原因如下**

如果一列中的多個任務被指派給相同的工作角色/個人，這表示這些任務可以合併。 將太多工指派給使用者可能會讓他們覺得有更多工作需要完成，這可能會影響Workfront的採用。

</br>
</br>

**最佳實務**

請確定範本任務包含任務持續時間、計畫時數和前置任務。

**原因如下**

這三個專案（持續時間、計畫時數和前置任務）是專案時間表的建置組塊。 這些是瞭解工作需要多久以及需要完成時間的關鍵。 Workfront的資源管理工具期間與計畫時數，加上工作角色指派，可計算資源產能、可用性等。

如果您不確定如何首次估計持續時間或計畫時數，請與專案團隊合作以定義一些初始估計。 使用範本後，請再次與專案團隊會面，以確定可以在何處進行變更以使範本更準確。 如果使用者在Workfront中記錄時間，您可以將專案的計畫時數與實際時數進行比較，以檢視需要調整的位置。


</br>
</br>

**最佳實務**

預先設定專案詳細資訊，並在範本上附加自訂表單。

**原因如下**

請確定已在專案範本中填入所有專案的標準資訊。 這不僅有助於加速專案的建立，還可以確保資訊的必要性並跨專案保持一致。

使用範本將請求轉換為專案時，附加符合請求自訂表單的專案自訂表單，以提取提交的資訊。

</br>
</br>

**最佳實務**

定期檢閱和更新專案範本。

**原因如下**

隨著流程和團隊的變更，應更新專案範本。 建立定期節奏（例如每季），以檢查並檢視哪些範本未主動使用。 您可以停用這些專案，因為它們仍在Workfront中，但不會出現在範本選擇清單中。

</br>
</br>

**最佳實務**

檢查範本，確保它們擁有所有需要的資訊，然後您才能共用範本，讓其他人開始使用。


**原因如下**

由於範本將一再用於製作專案，因此您需確定所有專案皆已正確且完整設定。 這可讓所有專案保持一致，並產生準確的報告資料。

除了工作設定（如持續期間和計畫時數）之外，共用範本前也要檢閱以下幾項：

* 排程開始日期設定
* 任務限制
* 專案所有者、贊助者、群組和公司
* Portfolio與方案
* 里程碑路徑和步驟
* 核准流程
* 確保指派給專案任務的使用者擁有專案的Contribute存取權
* 資源集區
* 提醒通知
* 排程
* 貨幣設定（如果適用）
* 附加標準檔案
* 附加必要的海關表格
* 檢查專案團隊，確認沒有指派額外的人員

</br>
</br>

**最佳實務**

當您調整範本上的「排程起始日期」選項時，請複查並更新任務限制。

**原因如下**

在專案上混合不同的任務限制可能會導致計畫日期計算出現意外和混淆。 例如，為「排程起始日期」選項選取「開始日期」時，預設會指派給在該專案中建立的任何任務儘快任務限制。 如果您稍後將「排程起始日期」選項切換為「完成日期」，則任何已建立的工作都會預設有「儘可能晚些」工作限制。 無意間將這些任務與每個限制條件混合在一起，可能會導致專案時間表中的計畫日期變得混亂。


</br>
</br>

**最佳實務**

檢查範本上的專案團隊，並移除不會使用專案的使用者。

**原因如下**

當您從現有專案建立範本時，它會帶出專案中被指派任務/問題的人員。 而且，當您使用範本時，可以移除先前被指派工作的人員或變更您在範本上自行作出的指派。

所有這些使用者都會列在專案的「人員」和「排程」區段中，作為專案團隊的一部分。 因此，這些範本會被傳播到從該範本建立的所有專案。 這可能會導致使用者混淆，因為身為專案團隊的一員，他們收到有關專案活動的通知、在「我參與的專案」清單中檢視專案，以及取得專案及其任務、問題和檔案的許可權。
