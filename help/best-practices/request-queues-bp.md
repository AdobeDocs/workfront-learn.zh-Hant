# 最佳實務 — 要求佇列

## 什麼是Adobe Workfront的「最佳實務」？

最佳做法是代表有效、有效行動方針的准則；易於被您和公司用戶採納；並可在貴組織中成功複製。

在您檢閱這些建議時，請記住，有些Workfront最佳實務是通用的，有些則可能是更具體的主題。 將這些最佳實務作為架構，協助您指導Workfront系統的設定與使用。

## 瀏覽此頁面

當您捲動此頁面時，首先會找到主題所有最佳實務的高階清單。 這可讓您檢閱建議，而不需深入了解「原因」的詳細資訊。

「這些最佳實務為何？」 在高階清單後找到的「 」區域，會更詳細說明某些最佳實務，以及為何這些作法被視為程式、工具等，您應考慮使用Workfront執行個體實作。

</br>
</br>

## 請求佇列最佳實務

* 包括請求隊列的每個元素的說明 — 請求隊列項目、主題組、隊列主題和路由規則。

* 建立一個名為「請求隊列」或「操作」的項目狀態，該狀態相當於「當前」，以便將請求隊列項目與其他項目區分開。

* 如果您打算對透過佇列提交的請求使用核發核准，請建立名為「已拒絕」的核發狀態。

* 將「通用」自訂表單指派給請求佇列，以盡可能擷取企業範圍內一致的資料。

* 避免與「所有人」共用請求佇列。 設定隊列詳細資訊設定，使用戶只能查看與其需求相關的隊列。

* 建立並指派包含請求佇列報表的控制面板，讓流量管理員、系統管理員或指派的使用者都能直接處理問題。

* 使用佈局模板從項目的左側面板菜單中為不需要建立隊列的用戶刪除請求隊列設定選項。

* 建立系統管理員請求佇列，讓使用者詢問Workfront相關問題、提出有關係統設定的請求、排程新的使用者培訓等。

* 定期審核請求隊列，以識別和取消共用未使用的隊列。

* 使用主題群組來組織請求佇列中超過10個佇列主題，以建立更短、更輕鬆的清單。

* 通過使用主題組和隊列主題劃分請求隊列，而不是建立多個隊列，控制用戶可用的請求隊列總數。

* 為每個隊列主題設定路由規則。 至少要設定預設的路由規則。

* 需要選擇性路由時，請利用主題組和隊列主題。

* 將請求路由至團隊，而非個人。

## 這些最佳實務為何？


**最佳實務**

包括請求隊列的每個元素的說明 — 請求隊列項目、主題組、隊列主題和路由規則。

**原因如下**

說明可讓群組管理員、未來的系統管理員或維護請求佇列的其他人確切知道請求佇列的每個片段的功能。

當您將游標移至新請求視窗中欄位上的資訊圖示上時，也會顯示說明資訊。

說明不必太長，只需簡短說明元素的用途或用途。

</br>
</br>

**最佳實務**

建立一個名為「請求隊列」或「操作」的項目狀態，該狀態相當於「當前」，以便將請求隊列項目與其他項目區分開。

**原因如下**

請求隊列在項目中「lives」，且狀態必須等於「當前」，隊列才會處於活動狀態。

為了將請求與具有「當前」狀態的實際工作項目區分開來，請建立一個狀態，以僅用於名為「請求隊列」或「操作」的請求隊列。 然後，您就可以使用此狀態，在撰寫報表時協助排除或納入請求佇列專案。

</br>
</br>

**最佳實務**

使用問題批准時，建立名為「已拒絕」的問題狀態，並將「如果已拒絕」選項設為「已拒絕」狀態。

**原因如下**

使用「已拒絕」狀態，即可清楚指出請求已審核和拒絕。

</br>
</br>

**最佳實務**

將「通用」自訂表單指派給請求佇列，以盡可能擷取企業範圍內一致的資料。

**原因如下**

「通用」自訂表單會收集要求所需的標準資訊，無論提交的要求類型為何。

擁有「通用」自訂表單有助於減少您建立和維護所需的自訂表單數量。 此外，還可確保所有請求都以相同的方式收集相同的資訊，進而產生一致的報表和資料分析。

</br>
</br>

**最佳實務**

避免與「所有人」共用請求佇列。  設定隊列詳細資訊設定，使用戶只能查看與其需求相關的隊列。

**原因如下**

在大多數情況下，請求佇列只需要與特定一組人員共用，例如團隊、廠商、客戶等。 當請求者在請求佇列清單中只看到所需項目時，便可輕鬆找到並導覽。

</br>
</br>


**最佳實務**

建立並指派包含請求佇列報表的控制面板，讓流量管理員、系統管理員或指派的使用者都能直接處理問題。

**原因如下**

讓使用者快速且輕鬆地存取傳入的請求，即表示工作不會在洗牌中丟失。

</br>
</br>

**最佳實務**

使用佈局模板從項目的左側面板菜單中為不需要建立隊列的用戶刪除請求隊列設定選項。


**原因如下**

這可確保所有請求隊列都經過正確的建立過程（例如由管理委員會審核），並由系統或組管理員正確設定。

此外，這有助於讓佇列清單保持井然有序，並著重於貴組織需要的請求類型。

</br>
</br>

**最佳實務**

建立系統管理員請求佇列，讓使用者詢問Workfront相關問題、提出有關係統設定的請求、排程新的使用者培訓等。

**原因如下**

這可讓使用者集中提交問題，讓管理員收集、監控及回應Workfront相關問題。

此外，此資訊可用於顯示系統管理員角色的時間、工作和價值，並作為附加系統管理員的理由。

</br>
</br>


**最佳實務**

定期審核請求隊列，以識別和取消共用未使用的隊列。

**原因如下**

定期稽核Adobe Workfront系統中的設定和項目，有助於使其保持簡潔，並避免不需要的項目。 如果不再使用或監視隊列，請確保用戶不再能訪問該隊列，以便工作請求不會陷入無效。

</br>
</br>


**最佳實務**

使用主題群組來組織請求佇列中超過10個佇列主題，以建立更短、更輕鬆的清單。

**原因如下**

主題群組會減少要選取的選項初始清單，以增加使用者採用率，減少混淆。 這可讓使用者在嘗試提交請求時輕鬆找到所要的內容，而無需壓倒他們。

此外，它使系統管理員和/或請求隊列管理員有機會為用戶建立平滑的導航路徑，並以更好的方式組織和報告提交的請求類型。

</br>
</br>

**最佳實務**

通過使用主題組和隊列主題劃分請求隊列，而不是建立多個隊列，控制用戶可用的請求隊列總數。

**原因如下**

過多的請求隊列使用戶很難找到所需內容。

隊列數減少也有助於流量協調員、系統管理員或管理隊列的其他人員，使他們能夠更快地找到所需的資訊，而無需導航到多個請求隊列項目。

如果不同請求隊列需要不同的訪問，或者合併隊列會令用戶困惑，請建立多個請求隊列。

</br>
</br>

**最佳實務**

為每個隊列主題設定路由規則。 至少要設定預設的路由規則。

**原因如下**

路由規則確保將始終為某人分配傳入的請求，以便工作不會漏掉。

</br>
</br>

**最佳實務**

需要選擇性路由時，請利用主題組和隊列主題。

**原因如下**

路由規則不能應用於自定義表單的欄位。 因此，如果需要將不同類型的請求路由到不同的團隊/個人，則使每種類型的請求都具有自己的主題組/隊列主題，以便能夠正確路由工作。

</br>
</br>

**最佳實務**

將請求路由至團隊，而非個人。

**原因如下**

當請求傳送給團隊時，它可讓整個團隊洞察所提出的請求，以及未來工作可能包含哪些內容。 每個人都可以查看「團隊」頁面中是否有新項目，或透過控制面板上的報表追蹤新項目。

它還確保當流量管理員或負責審核或分配傳入請求的其他人員不可用時，自動提供備份並有權訪問請求資訊。
