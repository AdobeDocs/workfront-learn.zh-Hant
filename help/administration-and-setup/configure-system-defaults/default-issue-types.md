# 了解預設問題狀態

因為每個問題類型都追蹤不同類型的資訊，每種類型可能需要唯一的工作流程，才能加以解決。 自訂每個問題類型的狀態，以符合該問題類型的工作流程。

## 預設狀態

有四種無法刪除的預設問題狀態：

* 新增
* 進行中
* 已關閉
* 保留

另外6種常見的問題管理狀態 [!DNL Workfront]:

* 重新打開
* 等待反饋
* 無法複製
* 已解決
* 確認完成
* 無法解決
